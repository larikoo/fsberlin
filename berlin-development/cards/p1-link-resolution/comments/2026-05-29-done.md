`berlin_core::links::resolve`: walks + parses a project, builds a project-
unique `slug → Card` index, and reports structural `LinkProblem`s:
- `Dangling` — a `depends_on`/`blocks`/`linked`/`criteria` slug resolves to
  no card (names the from-card and field);
- `DuplicateSlug` — a slug appears on more than one card (ADR-010 uniqueness);
- `WaypointInCriteria` — a waypoint slug in any `criteria` list (the ADR-011
  acyclicity guardrail).

Model gained `depends_on()/blocks()/linked()/criteria()` accessors on `Card`.
Parse/I-O failures are `Err`; structural problems are collected so a caller
(p1-cli-validate) can report them all at once. `supersedes`/`superseded_by`
are clause refs (ADR-012), not card relations, so they are intentionally not
resolved here.

Verified (Rust 1.95.0): 4 unit tests (sound graph, dangling, waypoint-in-
criteria, duplicate slug) + 2 integration tests; fmt clean; clippy
--all-targets -D warnings clean. The integration test proves both the example
and the full dogfooding project (≥25 cards) resolve with **zero** problems —
so "sound" isn't vacuous.
